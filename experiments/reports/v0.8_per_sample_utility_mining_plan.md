# BitDPM v0.8: Per-Sample Utility Mining and Capacity Expansion

## Status

Code scaffold completed. No heavy training or evaluation was started by Codex.

## Objective

v0.8 targets the bottleneck identified in v0.7:

> non-baseline oracle coverage remains sparse.

The goal is to expand coverage through:

1. a larger benchmark,
2. per-sample utility mining,
3. rank/capacity scan around the strongest v0.7 structure.

## Added Code

- `bitdpm/eval/v08_benchmark.py`
  - Adds a 100-sample expanded benchmark.
  - Categories: commonsense, math, code, Chinese, reasoning.
- `scripts/run_v05_router_calibration.py`
  - Adds `--benchmark-set core|v08`.
  - Keeps historical 45-sample benchmark as `core`.
  - Uses `v08` for the expanded 100-sample set.
  - Adds stable sampling support from the v0.7 fix.
- `scripts/mine_v08_utility.py`
  - Mines baseline/block relationships from eval JSON.
  - Outputs bucketed utility samples and training text candidates.
- `scripts/make_v08_capacity_scan.py`
  - Generates rank/scale scan commands for error-type `l22_l24_down`.

## Recommended First v0.8 Run

Start with rank 8 on the expanded benchmark to compare v0.7 structure under the new 100-sample set.

Train:

```bash
BITDPM_FORCE_CPU=1 python scripts/train_v07_blocks.py \
  --model Qwen/Qwen2.5-0.5B-Instruct \
  --output-dir experiments/outputs/blocks_v08_error_l22_l24_down_rank8 \
  --datasets calculation_error commonsense_choice format_following chinese_semantic short_reasoning \
  --structure l22_l24_down \
  --rank 8 \
  --epochs 5 \
  --batch-size 1 \
  --lr 2e-4 \
  --max-length 96
```

Evaluate deterministic at scale 0.60:

```bash
BITDPM_FORCE_CPU=1 python scripts/run_v05_router_calibration.py \
  --model Qwen/Qwen2.5-0.5B-Instruct \
  --load-blocks experiments/outputs/blocks_v08_error_l22_l24_down_rank8 \
  --benchmark-set v08 \
  --deterministic \
  --save-outputs \
  --block-scale 0.60 \
  --tag v08_rank8_scale060_v08_det
```

Evaluate stable sampling at scale 0.75:

```bash
BITDPM_FORCE_CPU=1 python scripts/run_v05_router_calibration.py \
  --model Qwen/Qwen2.5-0.5B-Instruct \
  --load-blocks experiments/outputs/blocks_v08_error_l22_l24_down_rank8 \
  --benchmark-set v08 \
  --seed 0 \
  --stable-sampling-seeds \
  --save-outputs \
  --block-scale 0.75 \
  --tag v08_rank8_scale075_v08_stable_sampling
```

Then mine the stable sampling report:

```bash
python scripts/mine_v08_utility.py \
  --report experiments/reports/<REPORT_NAME>.json \
  --tag v08_rank8_scale075_stable_sampling
```

## Success Criteria

Minimum:

- non-baseline oracle coverage increases beyond v0.7's `2/45` when normalized against the larger benchmark.
- deterministic best fixed remains at or above the v0.7 level.
- Always-All remains below selective/oracle.

Target:

- non-baseline oracle coverage reaches roughly 6-8 corrected samples on the expanded benchmark.
- stable sampling oracle exceeds the v0.7 `0.867` reference on comparable settings.
