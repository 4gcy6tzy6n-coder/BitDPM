# BitDPM AAAI Main Experiment Protocol

This protocol turns the current mechanism evidence into the next executable
main-result path. It does not introduce a new mechanism. It validates whether
the current admitted block pool and conservative router remain useful at larger
scale.

Use the current admitted pool unless a provenance-complete replacement is
trained:

```text
Registry: configs/bitdpm_v11_admitted_pool.json

Decoding: registry protocol
Router candidate: allow-core-no-log
Router features: has_multiplication, has_distance, has_coordinate, has_mean
```

cross-validates the conservative allow-core-no-log router, bootstraps CIs, and

## Success Gates

Minimum:

- v1k_clean oracle improves over baseline.
- v1k_clean Always-All is below oracle and treated as a negative control.
- Router net is positive on v14 or v15 and non-negative on v1k_clean.
- Breaks are lower than fixes.

Strong AAAI path:

- v1k_clean router net is positive with low breaks.
- v14 and v15 reproduce current zero-break conservative-router behavior.
- Bootstrap CI for router delta is non-negative or clearly positive on at least
  two benchmarks.
- External baselines are added.

The paper still needs this table before a high-confidence AAAI claim:

Prompt-only commands are generated by:

```bash

```

The prompt-only baseline uses `scripts/run_prompt_only_baseline.py` and the same
benchmark/evaluator path as BitDPM.

```bash
python scripts/make_lora_baseline_commands.py
pip install -e '.[peft]'
bash experiments/reports/lora_baseline_commands.sh
```

The LoRA baseline uses `scripts/run_lora_baseline.py`, PEFT LoRA, and the same
benchmark/evaluator path as BitDPM. Its outputs are auto-ingested by
`scripts/build_aaai_baseline_gap_table.py`.

## Claim Gate After v1k_clean

If v1k_clean has positive oracle but router is weak:

> Claim sparse correction utility, not deployable routing success.

If v1k_clean router is positive with low breaks:

> Claim conservative deployable routing has preliminary held-out support.

If v1k_clean router breaks many baseline-correct samples:

> Return to router safety calibration; do not expand block mechanisms.

If v1k_clean oracle coverage collapses:

> Current block pool is not broad enough for AAAI main result; train a
> provenance-complete main block/pool before submission.
